# テスト方針

## バックエンド

- ユニットテスト: `services` の業務判断を中心にテストする。
- DB を使うテストは、必要になった時点でテスト用 DB を用意して分ける。
- 小さいテンプレートでは `tests/` 直下に置いてよい。
- テスト関数名は `test_<対象>_<条件>_<期待結果>` の形式にする。
- フィクスチャは `conftest.py` に共通化する。

## フロントエンド

- ユニットテストを追加する場合は Vitest を使う。
- E2Eテストを追加する場合は Playwright を使い、`frontend/e2e/` に配置する。
- テストファイルは対象ファイルと同階層に `*.test.ts(x)` として配置する。
